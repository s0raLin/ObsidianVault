## 尽管 LLM 很强大，但也可能误导你

大型语言模型（LLM）非常适合执行诸如**生成文本或代码**之类的任务。不过，尽管它们有很多优点，有时 **LLM 也可能提供误导性的信息**。

---

# 知识截止（Knowledge Cut-offs）

像 **GPT-3** 和 **Claude** 这样的语言模型都有 **知识截止时间（knowledge cut-off）**，这意味着它们**不知道在最后一次更新之后发生的事情**。

这在**软件开发领域**尤其是一个挑战，因为新的语法、框架和库发展非常快，如果使用了**过时的代码**，可能会导致错误。

为了解决这个问题，OpenAI 提供的 **ChatGPT-4（付费版本）** 引入了 **“Browse with Bing（使用 Bing 浏览）”** 功能。  
这个功能允许 ChatGPT-4 **直接搜索互联网**，从而回答需要最新信息的问题。

随着 **GPT-4o** 的发布，**网页搜索功能已经默认集成**，确保用户可以获得**最新的数据**。

其他一些 LLM，例如：

- **Perplexity AI**
- **Google Gemini**

也能够通过**网络搜索**获取最新信息。

---

# 幻觉（Hallucinations）

**幻觉（Hallucination）** 指的是：  
**语言模型在回答问题时给出了错误或虚构的信息。**

例如，在下面的提示中，你可能会询问 **Eigen 模板库中复数向量叉积的实现**：

> 请提供一个代码示例，在 Eigen 模板库中实现复数向量的叉积。

然而，在查阅 **Eigen 官方文档**后发现：

**Eigen 实际上并没有这个实现。**

这说明 **LLM 有可能“编造”出看起来合理但实际上不存在的内容**。

依赖 AI 模型的信息可能会导致：

- 不准确的信息
- 错误的决策
- 对关键问题的误解

由于 AI 生成内容在各个行业中越来越普遍，**人工验证信息的准确性变得非常重要**。

### 如何减少幻觉

可以通过 **交叉验证（Cross-verification）** 和 **事实核查（Fact-checking）** 来减少幻觉：

方法包括：

- 将 AI 的回答与**官方文档**进行比较
- 查阅**权威资料**
- 进行**手动验证**

这样可以确保信息**可靠可信**，避免在**错误的基础上继续开发或决策**。

---

# 输入与输出限制（Input and Output Limitations）

模型根据**输入内容**来生成**输出结果**。

LLM 使用一种叫 **Tokenization（分词）** 的处理方式：

输入文本会被拆分成多个**Token（标记）**，模型通过这些 token 来理解和处理内容。

例如：

在某些情况下：

- 每个**单词**
- 每个**逗号**

都可能被计算为 **1 个 token**。

如果计算**字符数**：

- **空格**
- **单词**
- **标点符号**

都会被计算在内。

---

## Token 大致换算（OpenAI估算）

OpenAI 给出的粗略估计：

- **1 token ≈ 4 个英文字符**
- **1 token ≈ ¾ 个单词**
- **100 tokens ≈ 75 个单词**

---

## 上下文窗口（Context Window）

LLM 的输入 token 数量是有限制的。

例如：

|模型|上下文窗口|
|---|---|
|ChatGPT-3.5|4096 tokens|
|ChatGPT-4|8192 tokens|
|ChatGPT-4o|128k tokens|

更大的上下文窗口意味着：

- 可以处理**更多输入内容**
- 保持更好的**上下文理解能力**

---

⚠️ 注意：

AI 模型的 **价格和 token 限制可能会随着技术发展而改变**。  
建议查看 **OpenAI 官方定价页面**获取最新信息。

---

## Token 限制带来的问题

如果输入的 token **超过模型的限制**：

可能会导致：

- 模型**忘记之前的内容**
    
- 回答**不连贯**
    
- 输出**被截断**
    

LLM 依赖上下文窗口来保持回答的连贯性。

因此可以说：

**LLM 只有“短期记忆”**。

也就是说：

它只记得**当前上下文窗口里的内容**。

如果某些重要信息出现在**更早的对话中**，用户可能需要：

👉 **重新提醒模型这些信息**

---

## Token 计算工具

不同的语言模型**计算 token 的方式不同**。

为了估算 prompt 中 token 的数量，可以使用 OpenAI 提供的工具：

**Tokenizer**

它可以帮助你计算：

- 输入 prompt 的 token 数量
    
- 是否接近模型的限制
    

从而提高生成结果的质量。

---

# 总结（Conclusion）

为了有效使用 LLM，用户需要了解它们的**优点和局限性**。

使用时需要注意：

1. 控制输入的 **token 数量**  
2. 避免超过模型的 **上下文限制**

否则可能导致：

- 输出被截断
- 回答不完整

虽然 LLM 通常会生成**逻辑合理的回答**，但仍然需要：

✔ **进行事实核查**  
✔ **交叉验证信息**

因为模型可能会出现：

- **幻觉（hallucination）**
- **知识截止（knowledge cut-off）**

记住这些简单的原则，可以在使用生成式 AI 时 **节省大量时间和精力**。
