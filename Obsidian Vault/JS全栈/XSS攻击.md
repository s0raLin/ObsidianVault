## 什么是XSS攻击
XSS(跨站脚本)攻击，攻击者将恶意代码注入网页。当用户打开网页时，恶意代码在用户电脑中执行，并建立与攻击者的连接

XSS攻击主要用于窃取用户的cookie

## XSS攻击类型
XSS攻击通常分为两类
* **反射型XSS**
	当用户点击精心准备的链接会触发攻击
* **存储型XSS**
	当攻击者成功将代码注入服务器中时，每次访问原始界面，该恶意代码都会在浏览器中执行
有时还有一种**DOM型XSS**，发生在Web应用不安全处理不可信来源的客户端JS代码时。通常通过将数据写回DOM来实现。这类漏洞发生在客户端而非服务器

## 如何发现XSS漏洞
查看是否有用户可以影响网站内容的地方
比如检查用户输入的文本是否显示在页面上
```html
<script>alert("This web page is vulnerable")</script>
```

## 如何防御XSS攻击
有一些专门的服务可以帮助检测: https://xss-scanner.com/

此外，以下是开发者可以采取的一些 XSS 防御措施：

- 对用户输入进行 **编码**。
- 如果编码不适用，则使用 **验证** 替代。
- 在 **服务器端和客户端** 代码中都安全处理数据。
- 使用 HTTPOnly cookie 标志，它禁止 JavaScript 读取和写入 cookie。
- 对网页链接进行限制，仅允许白名单协议（如 `http://`、`https://`）。
- 使用 **内容安全策略（Content Security Policy, CSP）**。

从客户端防御角度，用户默认浏览器无法过滤恶意代码，因此会执行收到的任何脚本。要防御 XSS 攻击，可以：

- 定期将浏览器更新到最新版本
- 使用特殊浏览器扩展，如 Chrome 的 **NoScript**
