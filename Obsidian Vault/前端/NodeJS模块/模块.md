## 什么是模块?
可以把模块理解为一组你希望在应用中使用的函数。模块通常通过接口集成到软件中。

Node.js 中包含三种类型的模块：

- **核心模块（Core modules）**
- **本地模块（Local modules）**
- **第三方模块（Third-party modules）**


## 核心模块
Node.js提供一组核心模块，无需额外安装即可直接使用，要在Node.js应用中引入模块，可以使用`require()`函数，并在括号内传入模块名。这些模块会在Node.js进程启动时自动加载。

核心模块可以使用`node:`前缀标识，这样可以绕过require缓存。例如，require('node:http')将始终返回内置的HTTP模块，即使`require.cache`已经存在同名条目

核心模块之一是 **FS 模块**，它包含用于文件 I/O（输入/输出）的类、方法和事件

来看下面的代码片段

```js
const fs = require("node:fs");
```

**代码说明：**  
通过这种方式，你的应用就可以访问 FS 模块，并使用其提供的功能。

Node.js 中一些常见的核心模块包括：
- **HTTP** —— 让 Node.js 充当 HTTP 服务器
- **OS** —— 提供有关操作系统的信息
- **URL** —— 用于解析 URL 字符串
- **DNS** —— 用于执行 DNS 查询和名称解析功能

## 本地模块

你也可以创建自己的模块，并将它们作为本地模块轻松引入到应用中。

下面我们来创建一个向控制台输出问候信息的模块：
```js
exports.sayHello = function (name) {
  console.log("Hello", name);
};
```

**代码说明：**  
使用 `exports` 关键字可以将属性和方法暴露给模块外部使用。

将上述代码保存到名为 **`greetingModule.js`** 的文件中。

现在，你就可以在任意 Node.js 文件中引入并使用该模块了：
```js
const greeting = require("./greetingModule");
```
**代码说明：**  
注意这里使用了 `./` 来定位模块，表示该模块位于当前 Node.js 文件所在的同一目录中。

## 第三方模块

你可以借助 **Node 包管理器（Node Package Manager，简称 NPM）** 来安装第三方模块。NPM 是一个命令行工具，可用于安装、更新和卸载 Node.js 包。同时，它也是一个用于托管开源 Node.js 包的在线仓库。

NPM 会在安装 Node.js 时默认一并安装。要验证 NPM 是否安装成功，可以在终端中输入以下命令：
```js
npm -v
```
**代码说明：**  
该命令用于查看当前安装的 NPM 版本。

下面我们通过安装 **express** 模块来看看如何使用 NPM 安装模块：
```js
npm install express
```
**代码说明：**  
该命令会将 express 模块安装到当前项目中。

安装完成后，你就可以在项目中使用 `require()` 方法引入并使用 express 模块了。
