## 什么是包管理器?
一种用于安装和使用软件包的程序或系统，任何一个包在根目录都会包含一个配置文件，用于描述:
* 当前包所需的依赖
* 包名
* 作者信息
* 版本号等

## 详情
### 安装一个包
```bash
$ pnpm init -y && pnpm install express
```

#### 代码说明
- `pnpm init -y`  
    创建一个新的包，并自动生成默认配置
- `&&`  
    表示 **AND 操作符**，前一个命令成功后才执行后一个
- `pnpm install express`  
    从 npm 官方仓库安装 `express` 包



安装完成后，打开 `package.json` 文件，你会看到类似下面的内容：

```js
{
  "name": "what-is-package-managers",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "dependencies": {
    "express": "^4.18.2"
  }
}
```


### 代码说明

这个文件包含：

- `name`：包名
- `version`：版本号
- `description`：描述
- `dependencies`：当前项目所依赖的所有包（这里只有 `express`）

如果你想查看 **express 的全部依赖**，可以查看 `package-lock.json` 文件，其中记录了完整的依赖树。


## 包管理器的类型

Node.js 中常见的包管理器有：

- **npm**  
    Node.js 自带的标准包管理器（Node Package Manager）
- **Yarn**  
    Facebook 开发的包管理器
- **pnpm**  
    新一代包管理器，速度更快、磁盘占用更小

## 包的元信息

包管理器允许开发者为包定义：
- 名称
- 描述
- 版本号
- 作者信息
- 文档链接等

## 查找与安装

大多数包都存储在互联网上的**包仓库（registry）**中，而且不止一个：
- npm 有自己的官方仓库
- GitHub 也提供包仓库

当你安装某个包时，它会从互联网下载到你的本地计算机。


## 依赖（Dependencies）

**依赖**是指一个包运行所需要的其他包。
但请注意：
- 每个依赖包**本身也有依赖**
- 这些依赖会层层嵌套
包管理器通过**锁文件**来记录完整的依赖关系。


## 锁文件（Lock File）

每一个 JavaScript 包管理器都有一个锁文件，例如：

- `package-lock.json`
- `yarn.lock`
- `pnpm-lock.yaml`

锁文件记录了：

- 所有依赖包
- 它们的**精确版本号**

你可以把锁文件提交到仓库中，这样在另一台机器上就能**完全复现**当前项目的依赖环境。

### 要不要提交锁文件？

- ✅ **应用项目（Web / 服务端）**：**应该提交**
    - 确保服务器与本地环境一致
- ⚠️ **工具库 / 公共库**：一般**不建议提交**

这并不是硬性规定，而是一种经验建议，具体仍需根据项目情况决定。
