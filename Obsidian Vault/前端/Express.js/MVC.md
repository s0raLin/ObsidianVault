**MVC**（Model-View-Controller，**模型-视图-控制器**）是一种架构模式，它将应用程序逻辑上分为三个主要部分：

- **Model（模型）**：负责所有数据及其相关逻辑；
- **View（视图）**：负责向用户呈现数据，或者处理用户的交互；
- **Controller（控制器）**：接收用户输入，并通知 Model 需要进行哪些更改。

这就是最基础的 MVC 架构流程。

> 还有一种常见的变体：在这种实现中，View 和 Controller 之间也存在直接联系，Controller 在 Model 和 View 之间起到桥梁和协调的作用。


### MVC 的优点与缺点

MVC 之所以成为非常受欢迎的模式，并被广泛使用，是因为它带来了很多实际好处：

**优点**：

- 通过将应用拆分成独立组件，大大提高了应用的**灵活性**、**可维护性**和**可扩展性**。
- 可以**单独测试**各个组件（Model、View、Controller）。
- 组件具有良好的**可重用性**。
- 一个 Model 可以对应**多个不同的 View**（同一份数据可以有多种展示方式）。
- 可以为不同组件配置**不同级别的安全性**。

**缺点**：

- 需要记住：**它不适合小型应用**。在简单项目中使用 MVC，反而会让代码变得更复杂、不必要的繁琐。
- MVC **不适合对性能要求极高的应用**。因为它引入了多层架构，有时直接穿过几层会更高效。


MVC 并不是唯一的架构模式。如果因为某些原因 MVC 不适合你的项目，还有其他类似目的的模式可以选择：MVP（Model-View-Presenter）、MVVM（Model-View-ViewModel） 等。这些模式常被统称为 *MV 家族**。
