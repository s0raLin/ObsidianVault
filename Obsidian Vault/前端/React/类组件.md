## 类组件结构
React中的类组件是ES6 JavaScript类，并且需要继承`React.Component`类。该组件的作用是封装UI元素的逻辑和状态
```jsx
import React from "react";

class App extends React.Component {
	render() {
		return <h1>Hello,World</h1>
	}
}

export default App;
```

## 构造函数和state
类组件的一个重要特性是能够保存和管理本地状态，通常通过构造函数`constructor`来实现
```jsx
import React, { Component } from "react";

class App extends Component {
  constructor(props) {
    super(props);
    this.state = {
      greeting: "Hello",
    };
  }

  render() {
    return <h1>{this.state.greeting}, World!</h1>;
  }
}

export default App;
```

在这个示例中，我们添加了一个构造函数，并在其中通过 `this.state` 初始化组件的状态。  
初始状态包含一个名为 `greeting` 的属性，值为 `"Hello"`。  
在 `render` 方法中，我们使用该状态来动态地显示问候语。

## 从函数组件转换为类组件
### 函数组件
```jsx
import React from 'react';

const MyFunctionlComponent = () => {
	return <h1>Hello, World</h1>;
}
```

### 类组件：

```jsx
import React from "react";
 
class App extends React.Component {
  render() {
    return <h1>Hello, I'm a class component!</h1>;
  }
}
 
export default App;
```


## 事件处理与 props 的使用

### 类组件中的事件处理

在类组件中处理事件，通常是通过定义一个方法来响应特定事件。例如，处理按钮点击事件：

```jsx
class App extends Component {
  handleClick = () => {
    console.log("Button clicked!");
  };

  render() {
    return <button onClick={this.handleClick}>Click me</button>;
  }
}
```


### 类组件中的 props 使用

在类组件中使用 props 同样非常直观，示例如下：

```jsx
class Greeting extends Component {
  render() {
    return <h1>{this.props.message}</h1>;
  }
}

// 使用方式
const App = () => <Greeting message="Hello, World!" />;
```
#### 代码解释

上述代码定义了一个名为 `Greeting` 的类组件。  
在 `render` 方法中，通过 `this.props.message` 获取传入的 `message` 属性，并将其显示在 `<h1>` 标签中。

在使用示例中，函数组件 `App` 渲染了 `Greeting` 组件，并向其传递了一个 `message` 属性，值为 `"Hello, World!"`。

这展示了 React 中 **props 的概念**：  
props 允许将数据从父组件（这里是 `App`）传递到子组件（`Greeting`），从而根据不同需求动态配置组件。


## 优点与缺点

类组件具有各自的优缺点，理解这些有助于你决定何时使用类组件，何时使用函数组件。

### 优点

- **本地状态管理**：类组件可以通过构造函数方便地管理本地 state
- **生命周期方法**：类组件提供了完整的生命周期方法，用于在组件生命周期的不同阶段执行特定操作

### 缺点

- **语法冗长**：相比函数组件，类组件通常需要更多的样板代码
- **复杂性较高**：随着应用规模的增长，类组件可能会变得更难维护
