## State 与 Props 的比较

现在你已经对 state 和 props 有了基本认识，让我们来重点看看它们的不同之处。

### Props 示例：UserProfile 组件
```jsx
function UserProfile(props) {
  return (
    <div>
      <h1>{props.name}</h1>
      <p>{props.email}</p>
    </div>
  );
}

// 使用方式
function ParentComponent() {
  return <UserProfile name="John Doe" email="john.doe@example.com" />;
}
```


### 代码说明

在这个示例中，`UserProfile` 组件通过 props 接收用户的姓名和邮箱信息，并将其展示在页面中。

---

### State 示例：ToggleSwitch 组件

```jsx
import React, { useState } from 'react';

const ToggleSwitch = () => {
  const [isToggled, setToggle] = useState(false);

  const handleToggle = () => {
    setToggle(!isToggled);
  }

  return (
    <div>
      <button onClick={handleToggle}>
        {isToggled ? 'ON' : 'OFF'}
      </button>
    </div>
  );
}

export default ToggleSwitch;
```

### 代码说明

在这个示例中，`ToggleSwitch` 组件通过 state 来控制自身的开关状态，并根据当前状态动态改变按钮文本。

---

## 所有权与控制权（Ownership and Control）

- **State**  
    State 由组件自身拥有和管理，用于保存和更新内部数据。例如，`ToggleSwitch` 组件管理自己的 `isToggled` 状态。
- **Props**  
    Props 由父组件定义，并传递给子组件。子组件**不能修改 props**。例如，`UserProfile` 从父组件接收 `name` 和 `email`。

---

## 可变性（Mutability）

- **State 是可变的**  
    State 表示可以随时间变化的数据，可以通过状态更新函数进行修改。在 `ToggleSwitch` 中，点击按钮会触发 `handleToggle`，从而改变 `isToggled` 的值。
- **Props 是只读的**  
    Props 不能在子组件中被修改，它们主要用于展示数据或传递行为。`UserProfile` 只能使用 `name` 和 `email`，但不能改变它们的值。

---

## 重新渲染（Re-rendering）

- **State 的变化会触发组件重新渲染**  
    这使得 UI 能够反映最新的状态。在 `ToggleSwitch` 示例中，每次状态变化都会导致组件重新渲染，因此按钮文字会随之更新。
- **Props 的变化也会触发子组件重新渲染**  
    当父组件传入新的 props 时，子组件会使用最新的 props 进行重新渲染。

---

## 使用场景（Usage）

- **State**  
    适合用于组件内部的、无需与其他组件共享的数据。
- **Props**  
    用于从父组件向子组件传递数据或行为，使组件更具复用性。

---

## State 和 Props 的应用场景

### State 的常见用途：

- 管理表单输入（如输入框的值）
- 控制 UI 切换（显示 / 隐藏组件）
- 存储组件专属数据（如计数器值、选中项）
- 保存 API 请求返回的数据，并在数据更新时触发重新渲染
- 用于条件渲染，根据状态决定显示哪些组件或元素

### Props 的常见用途：

- 向子组件传递数据
- 控制组件的外观或行为，使组件可以在不同场景下复用
- 通过传递回调函数，实现父子组件甚至兄弟组件之间的通信

需要记住的是，这些只是常见示例。**State 和 props 往往会协同工作**，用于构建复杂且交互丰富的 React 应用
