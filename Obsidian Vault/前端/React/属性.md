在React中，**props(properties,属性)** 可以将数据从父组件传递给子组件
这些props是只读的，子组件不能修改

props和HTML属性的区别是可以传递JavaScript值，例如:
* 对象(objects)
* 数组(arrays)
* 函数(functions)
* 甚至是其他组件(components)


## 熟知的属性
熟知的HTML属性在React中同样可以作为props使用，只是名称发生了一些小变化
例如: `className`、`src`、`width`、`height`

```jsx
function Picture() {
	return (
		<img
			className="pricture"
			src="https://example-img"
			alt="Example.img"
			width={100}
			height={100}
	)
}

export default Picture;
```

## 从父组件向子组件传递props
例如有一个`User`父组件，想向子组件`Profile`传递数据:
```js
function User() {
	const age = 25;
	const city = "New York";
	
	return (
		<Profile
			fullName={{name: 'John', surname: 'Doe'}}
			age={age}
			city={city}
	)
}
```
**代码说明**

`Profile`接受三个props
* fullName：一个包含name和surname字段的对象
* age和city：通过`{}`传递JavaScript变量：age={age}


## 在子组件中使用props
例如，在`Profile`组件中：
```jsx
function Profile(props) {
	return (
		<div>
			<h2>Name: {props.fullName.name} {props.fullName.surname}</h2>
			<p>Age: {props.age}</p>
			<p>City: {props.city}</p>
		</div>
	);
}
```

## 通过解构访问props
```jsx
function Profile({fullName, age, city}) {
	return (
		<div>
			<h2>Name: {fullName.name} {fullName.surname}</h2>
			<p>Age: {age}</p>
			<p>City: {city}</p>
	);
}
```

## 使用解构并设置默认props值

```jsx
function Profile({ fullName, age = 30, city = "Unknown" }) {
  return (
    <div>
      <h2>Name: {fullName.name} {fullName.surname}</h2>
      <p>Age: {age}</p>
      <p>City: {city}</p>
    </div>
  );
}

function User() {
  return (
    <Profile
      fullName={{ name: 'John', surname: 'Doe' }}
    />
  );
}
```
- `age = 30`、`city = "Unknown"` 是默认值
- 当父组件没有传入这些 props 时，默认值会生效
- `fullName` 没有默认值，因此是**必需的**

## 转发props
当需要向子组件传递大量props时，可以使用JSX展开语法
```jsx
function ParentComponent() {
  const commonProps = {
    prop1: "value1",
    prop2: "value2",
    prop3: "value3"
  };

  return (
    <ChildComponent {...commonProps} />
  );
}

function ChildComponent(props) {
  return (
    <div>
      <p>Prop 1: {props.prop1}</p>
      <p>Prop 2: {props.prop2}</p>
      <p>Prop 3: {props.prop3}</p>
    </div>
  );
}
```
### 代码说明

`{...commonProps}` 会把对象中的所有属性一次性传递给子组件。
⚠️ **注意**：  
如果你在很多地方频繁使用 props 展开，可能说明组件设计需要优化。  
此时可以考虑：

- 拆分组件
- 使用 `children` 传递 JSX

## 将组件作为props传递

```jsx
function App() {
  return (
    <div>
      <Card
        body={<CardBody content="Body here" />}
        footer={<CardFooter content="Footer here" />}
        header={<CardHeader content="Header here" />}
      />
    </div>
  );
}
```

```jsx
function Card({ body, footer, header }) {
  return (
    <div>
        {header}
        {body}
        {footer}
    </div>
  );
}
```

## 使用 children 传递 JSX

```jsx
function ParentComponent() {
  return (
    <div>
      <h2>Parent Component</h2>
      <ChildComponent>
        <p>This is JSX passed as a child.</p>
        <button>Click me</button>
      </ChildComponent>
    </div>
  );
}
```

```jsx
function ChildComponent(props) {
  return (
    <div>
      <h3>Child Component</h3>
      {props.children}
    </div>
  );
}
```
### 代码说明

- 父组件将 JSX 放在子组件标签内部
- 子组件通过 `props.children` 访问这些内容

这种方式非常适合构建通用布局组件（如 `Card`、`Modal`、`Layout` 等）。

## Props的不可变性
在React中的props是不可变的，一旦传入就不能修改
```jsx
const MyComponent = (props) => {
	props.name = 'john'; //无效
	return <div>Hello, {props.name}!</div>
}
```

### 代码说明

React 会忽略对 props 的修改。  
如果你需要改变数据：
- 使用 **state**
- 或者让父组件传入新的 props

这种设计保证了**单向数据流**，让应用更可预测、更易维护
