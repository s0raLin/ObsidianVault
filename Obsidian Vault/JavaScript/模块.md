## 什么是模块?
**模块(module)** 是一个包含独立代码的单个文件。
模块可以大大提升程序的可维护性:
* 当你想重写某个模块时，只需要修改它自身的代码，而不会影响程序的其他部分

## 模块的好处
模块可以:
* 封装代码
* 减少全局变量
* 避免重复代码
* 提高可维护性和可读性

## 导入和导出(Import&Export)
### 导出函数
如果你想在其他文件中使用某个函数，需要使用`export`关键字
```js
// library.js
export function myBestFunction() {
	//...
}
```

### 导入函数
在需要该函数的文件中，使用`import`导入
```js
// main.js
import {myBestFunction} from './library.js'

myBestFunction();
```

### 使用`as`重命名导入的函数
```js
import {myBestFunction as myFunctiion} from './library.js';

myFunction();
```

### 导入整个模块
如果一个模块中有很多内容，可以一次性导入全部
```js
import * as myLib ffrom './library.js';

myLib.myBestFunction();
```

### 默认导出(default export)
你也可以使用`default` 导出，这样导入时不需要使用花括号
```js
// library.js
export default function myBestFunction() {
	//...
}
```

```js
// main.js
import anyNameToFunction from './library.js';

anyNameToFunction();
```

### 同时使用默认导出和普通导出

```js
// library.js
export function myFunction() {
	// ...
}

export default function myBestFunction() {
	// ...
}

export class MyClass {
	// ...
}
```

```js
// main.js
import anyNameToFunction, {myFunction, MyClass} from './library.js';

anyNameToFunction();

const myInstance = new MyClass();

myFunction();
```

## 关于默认导出的争议

开发者之间对“是否应该默认使用 default export”存在争议。

问题在于：  
👉 每个程序员都可以给默认导入起不同的名字，容易导致代码风格混乱。

**推荐做法：**  
让导入的名字和文件名保持一致。
```js
// main.js
import library, { myFunction } from './library.js';

library();
myFunction();
```

这样可以提高代码的一致性和可读性。


## 模块不仅能导出函数

在实际项目中，模块不仅用于函数，还常用于：
- 类（class）
- 常量（constants）
- 配置对象
- 工具函数等


## 启用 ES 模块支持

在使用 ECMAScript 模块时，你可能会遇到兼容性问题。  
要在 `.js` 文件中启用 ES Modules，请确保你的 `package.json` 中包含：
```json
{
  "type": "module"
}
```
**解释：**  
这告诉 Node.js 使用 ES Module 语法，而不是旧的 CommonJS。
